# Forum AI Notetaker

A course-based platform for turning class recordings into searchable transcripts and structured notes. Professors create courses, invite students with a code, and any course member can upload an audio or video recording. The backend extracts audio with FFmpeg, transcribes it with OpenAI Whisper, and generates a summary, topic list, and action items with Groq. Everything is searchable across courses the user belongs to.

## What's in the app

**Accounts & courses**
- Email + password auth with JWT sessions
- Two account types at sign-up: `professor` and `student`
- Professors create courses; each course has a 6-character invite code
- Students join a course with the invite code
- Professors can promote students to `ta` inside a course

**Recordings**
- Upload `mp4`, `mp3`, `wav`, or `m4a` files scoped to a course
- Upload permitted to instructors and TAs of the course
- Background pipeline: audio extraction → transcription → note generation
- Non-blocking upload: the response returns immediately with a session id; the client polls for status
- Per-session status lifecycle: `uploaded → processing → transcribed → notes_generated`, with `notes_failed` and `failed` terminal states
- Interrupted sessions are recovered on backend startup (anything stuck in `processing` is re-queued)

**Transcripts & notes**
- Word-level segment data kept alongside the full transcript
- Notes generated by Groq (`llama-3.3-70b-versatile`) as `{summary, topics[], action_items[]}`
- Long transcripts are split into chunks sized for Groq's per-minute token budget; per-chunk notes are merged and deduplicated

**Discovery**
- Full-text search over session title, transcript content, and notes
- Results are always scoped to courses the user is enrolled in
- Snippets with match highlights

**Playback**
- Original recording streamed back via the backend with HTTP range support, so the `<audio>` / `<video>` element can seek

## Pipeline

```
Upload (mp4/mp3/wav/m4a)
      │
      ▼
FFmpeg → mono 16 kHz WAV
      │
      ▼
Whisper "base.en" → transcript + word segments  ──► SQLite: transcripts
      │
      ▼
Groq llama-3.3-70b → {summary, topics, action_items}  ──► SQLite: notes
```

If `GROQ_API_KEY` is missing or invalid, the transcript is still saved and the session lands in `notes_failed`. The Whisper `base.en` model is baked into the backend Docker image, so the first transcription does not require internet.

## Quick start (Docker)

One command brings up the full stack — Python, Node, FFmpeg, Whisper, and the React build are all handled for you.

```bash
cp .env.example .env          # optional: paste a Groq key into .env
docker compose up --build
```

- Frontend: http://localhost:5173
- Backend API: http://localhost:5000

The Groq API key is **provided separately** with the assignment submission — no signup needed. Without a key, uploads still transcribe successfully; only note generation is skipped.

### First-time walkthrough

The app has no anonymous mode: recordings always live inside a course. From the frontend:

1. Open http://localhost:5173 and **register a professor account** (pick "Professor" on the sign-up form).
2. On the dashboard, click **Create a course**. Share the invite code with students.
3. **Register a student account** (new email, pick "Student") and **Join a course** with the invite code.
4. As the professor or TA, open the course and **upload a recording**. The status badge updates through `uploaded → processing → transcribed → notes_generated`.
5. Click the session to view the transcript, notes, and original recording. Use **Search** from the navbar to look across every course you belong to.

## Architecture

```
┌──────────────────┐        ┌──────────────────────────────────┐
│  frontend (nginx)│        │          backend (gunicorn)      │
│  React build     │  /api  │  Flask + Whisper + FFmpeg        │
│  port 5173 → 80  │ ─────▶ │  port 5000                       │
└──────────────────┘        │                                  │
                            │  ./data            → /app/data   │
                            │  ./backend/uploads → uploads     │
                            └──────────────────────────────────┘
```

Two containers on the default Compose network. nginx serves the built React SPA and proxies `/api/*` to the backend by service name. The backend runs gunicorn with a single worker and a thread pool that owns the transcription/notes pipeline so uploads return immediately. SQLite and uploaded recordings are bind-mounted (`./data` and `./backend/uploads`) so data survives container restarts and rebuilds.

## Data model

Stored in a single SQLite database at `./data/forum_ai_notetaker.sqlite3`:

| Table            | Purpose                                                  |
| ---------------- | -------------------------------------------------------- |
| `users`          | email, name, password hash, `user_type` (student/professor) |
| `courses`        | name, 6-character `invite_code`                          |
| `course_members` | `(course_id, user_id, role)` with role ∈ student/ta/instructor |
| `sessions`       | one recording: title, filename, stored path, status, `course_id` |
| `transcripts`    | one per session: full text + JSON segments               |
| `notes`          | one per session: summary, topics, action items           |

Sessions cascade-delete transcripts and notes. Dropping a course sets its sessions' `course_id` to NULL rather than deleting the recordings.

## API reference

All endpoints under `/api` except `/api/auth/register` and `/api/auth/login` require `Authorization: Bearer <token>`. The one exception is `GET /api/sessions/<id>/media`, which also accepts `?token=<jwt>` in the query string so `<audio>` / `<video>` tags can stream with range requests.

Responses use a consistent envelope:

```json
{"success": true,  "message": "...", "data": { ... }}
{"success": false, "error":   "..."}
```

### Health

| Method | Endpoint      | Description         |
| ------ | ------------- | ------------------- |
| GET    | `/`           | API running message |
| GET    | `/api/health` | Health check        |

### Auth

| Method | Endpoint             | Description                             |
| ------ | -------------------- | --------------------------------------- |
| POST   | `/api/auth/register` | Create an account; returns JWT          |
| POST   | `/api/auth/login`    | Exchange email + password for a JWT     |
| GET    | `/api/auth/me`       | Identity of the authenticated user      |

### Courses

| Method | Endpoint                              | Description                                |
| ------ | ------------------------------------- | ------------------------------------------ |
| POST   | `/api/courses/`                       | Create a course (professors only)          |
| GET    | `/api/courses/`                       | List courses the user belongs to           |
| POST   | `/api/courses/join`                   | Join a course with an invite code          |
| GET    | `/api/courses/<id>`                   | Course details and members                 |
| GET    | `/api/courses/<id>/sessions`          | Sessions within a course                   |
| PATCH  | `/api/courses/<id>/members/<user_id>` | Promote / demote a member (instructor-only) |

### Sessions

| Method | Endpoint                    | Description                                |
| ------ | --------------------------- | ------------------------------------------ |
| GET    | `/api/sessions/`            | Sessions from every course the user is in  |
| GET    | `/api/sessions/search?q=`   | Full-text search across title/transcript/notes |
| GET    | `/api/sessions/<id>`        | One session                                |
| GET    | `/api/sessions/<id>/media`  | Stream the original recording (range-aware) |
| POST   | `/api/sessions/upload`      | Upload a recording to a course             |

### Transcripts & notes

| Method | Endpoint                         | Description                     |
| ------ | -------------------------------- | ------------------------------- |
| GET    | `/api/transcripts/<session_id>`  | Transcript text and segments    |
| GET    | `/api/notes/session/<id>`        | Generated notes for a session   |

### End-to-end example (curl)

```bash
# 1. Register a professor and capture the JWT
PROF_TOKEN=$(curl -s -X POST http://localhost:5000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"prof@example.com","password":"password123","name":"Prof","user_type":"professor"}' \
  | python3 -c "import sys, json; print(json.load(sys.stdin)['data']['token'])")

# 2. Create a course and capture its id
COURSE_ID=$(curl -s -X POST http://localhost:5000/api/courses/ \
  -H "Authorization: Bearer $PROF_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"Intro to CS"}' \
  | python3 -c "import sys, json; print(json.load(sys.stdin)['data']['id'])")

# 3. Upload a recording to that course (returns immediately)
curl -X POST http://localhost:5000/api/sessions/upload \
  -H "Authorization: Bearer $PROF_TOKEN" \
  -F "file=@recording.mp4" \
  -F "title=Lecture 1" \
  -F "course_id=$COURSE_ID"

# 4. Poll the session until status=notes_generated
curl -H "Authorization: Bearer $PROF_TOKEN" \
  http://localhost:5000/api/sessions/<session_id>
```

## Environment variables

| Variable        | Where           | Effect                                                         |
| --------------- | --------------- | -------------------------------------------------------------- |
| `GROQ_API_KEY`  | `.env`          | Enables Groq note generation. If empty, placeholder, or invalid, transcripts still save but notes are skipped (`notes_failed`). |

`.env.example` documents the default. The Compose file loads `.env` as optional, so an unset file is fine.

## Manual setup (without Docker)

Only needed if you want to run pieces locally for development.

**Prerequisites**

- Python 3.9+
- Node.js 20+
- FFmpeg (`brew install ffmpeg` on macOS, `apt-get install ffmpeg` on Debian-ish)

**Backend**

```bash
cd backend
python -m venv ../venv
source ../venv/bin/activate
pip install -r requirements.txt
python app.py
```

Runs at http://127.0.0.1:5000. Schema is auto-initialized and any sessions left in `processing` from a prior run are recovered.

**Frontend**

```bash
cd frontend
npm install
npm run dev
```

Runs the Vite dev server at http://localhost:5173. The dev server proxies `/api` to `http://localhost:5000` — start the backend first.

## Troubleshooting

**Whisper model download fails with SSL certificate error.** When running outside Docker on macOS Python, run the bundled certificate installer, then retry:

```bash
open "/Applications/Python 3.14/Install Certificates.command"
python -c "import whisper; whisper.load_model('base.en')"
```

Inside Docker this is a non-issue — the model is baked into the image.

**A session is stuck in `processing`.** Restart the backend. `recover_interrupted_processing_sessions` at startup re-queues anything still marked processing from a previous crash.

**Notes show `notes_failed`.** Groq was called but returned an error. Check `GROQ_API_KEY`, verify the key is valid for `llama-3.3-70b-versatile`, and look at the backend logs for the traceback. The transcript is still available under `/api/transcripts/<session_id>`.

**Upload returns `course_id is required`.** Every upload must include a `course_id` form field for a course the user can post to (instructor or TA).

**Upload returns 403 `You do not have permission to upload to this course`.** Only the course's professor or a TA can upload. Promote the user via `PATCH /api/courses/<id>/members/<user_id>` with body `{"role": "ta"}`.
